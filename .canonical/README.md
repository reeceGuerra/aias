# Canonical — templates, rules, subagents, and generation

This directory contains canonical templates, organized by artifact type, and the generation infrastructure used to produce platform-specific artifacts for focused workspaces.

```
aias/.canonical/
  modes/          — mode templates (9 total: 7 core + 2 transversal)
  rules/          — rule templates (base-rule.md, output-contract.md, continuous-improvement.mdc)
  subagents/      — framework canonical sub-agent definitions (6 review sub-agents)
  generation/     — generator script, CLI, and validation artifacts
```

## `modes/` — Mode Templates (Phase 2)

All 9 modes are template-based. Transversal modes (`delivery`, `devops`) have built-in default bindings and stack-profile overrides are optional.

| Template | Purpose |
|---|---|
| `planning.mdc` | Planning mode behavior |
| `dev.mdc` | Developer mode behavior |
| `qa.mdc` | QA mode behavior |
| `debug.mdc` | Debug mode behavior |
| `review.mdc` | Code review mode behavior |
| `product.mdc` | Product analysis mode behavior |
| `integration.mdc` | Integration mode behavior |
| `delivery.mdc` | Ticket readiness and release visibility. Activated with `@delivery`. |
| `devops.mdc` | CI/CD pipeline management. Activated with `@devops` (also auto-activates on `*.yml`/`*.yaml`). |

Generated outputs are committed to `aias-config/modes/`.

Commands (`/blueprint`, `/issue`, `/fix`, etc.) are advisory/operative skills and live in **aias/.skills/** (framework) and **aias-config/skills/** (project custom). They carry `disable-model-invocation: true` in their frontmatter.

## `rules/` — Canonical Rules (Phase 4)

3 canonical rule sources:

| Source | Purpose | Governing contract |
|---|---|---|
| `base-rule.md` | Template for `base.mdc` files | `aias/contracts/readme-base-rule.md` |
| `output-contract.md` | Template for `output-contract.mdc` files | `aias/contracts/readme-output-contract.md` |
| `continuous-improvement.mdc` | Transversal continuous improvement rule | — |

Rule templates use placeholders (`{{...}}`) resolved from stack profile bindings (`binding.rule.base.*` and `binding.rule.output_contract.*`).

## `subagents/` — Framework Canonical Sub-Agents

6 review sub-agent definitions (framework read-only):

| File | Role |
|---|---|
| `aias-correctness-reviewer.md` | Logic, edge cases, contract violations |
| `aias-quality-reviewer.md` | Clarity, naming, maintainability |
| `aias-architecture-reviewer.md` | Pattern conformance, layer boundaries |
| `aias-test-auditor.md` | Test coverage and assertion quality |
| `aias-security-auditor.md` | Security posture and threat surface |
| `aias-reflector.md` | Meta-review and cross-dimension synthesis |

**Three-tier projection model:**
- `aias/.canonical/subagents/` → framework defaults (this directory, read-only)
- `aias-config/subagents/` → project-owned copies; regenerated by `aias generate --shortcuts`
- `.cursor/agents/` → tool-specific symlinks pointing to `aias-config/subagents/`

`aias-config/subagents/` is a project-owned generated surface: editable by the adopter, but regenerable by tooling. Git is the mechanism for reviewing and accepting overwrites.

Governing contract: `aias/contracts/readme-multi-agent-review.md`.
